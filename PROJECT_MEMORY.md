# 项目记忆（git_binance_trader）

## 项目目标
- 仅使用模拟资金进行币安规则 1:1 复刻交易。
- 目标：稳定提升收益，控制全程最大回撤不超过 15%。
- 收益目标：日收益率 >= 1%（以风险约束为前提的优化目标，不承诺固定收益）。

## 刚性红线
- 禁止实盘：任何代码与配置不得触发真实资金下单。
- 风控阈值：
  - 全程最大回撤 <= 15%
  - 单日最大回撤 <= 5%
  - 单笔亏损 <= 账户总资金 1%
- 触发阈值后必须执行：全仓平仓、暂停交易、进入策略重构流程。
- 合规范围：仅允许交易币安市值前 300 币种。

## 运行基线
- 初始模拟资金：10000 USDT。
- 交易模式环境变量：`TRADING_MODE=SIMULATION`。
- 关键凭据仅允许通过环境变量加载，不得明文入库。

## 核心交付清单
- 交易核心：
  - 币安 API 对接。
  - 模拟撮合引擎（现货、USDT 永续、Alpha 套利）。
  - 自动下单/平仓/持仓管理与异常容错。
- 策略体系：
  - 7x24 行情与绩效监控。
  - 策略更新先回测后上线。
- 前端体系：
  - 实时展示净值、收益/回撤、交易明细、策略状态。
  - 一键全仓平仓、暂停交易入口。
- 运维部署：
  - fly.io 一键部署，Dockerfile 与 fly.toml 标准化。
  - 版本更新与异常回滚能力。
- 监控复盘：
  - 风控与系统告警。
  - 每日交易复盘报告与优化记录。

## 迭代与发布纪律
- 每次程序更新必须：
  - 递增 `VERSION`
  - 更新 `CHANGELOG.md`（中文说明）
  - 更新本文件（记录决策、风险、结果）
  - 完成后提交到 GitHub 仓库
