# 项目记忆（git_binance_trader）

## 项目目标
- 仅使用模拟资金进行币安规则 1:1 复刻交易。
- 目标：稳定提升收益，控制全程最大回撤不超过 15%。
- 收益目标：日收益率 >= 1%（以风险约束为前提的优化目标，不承诺固定收益）。

## 刚性红线
- 禁止实盘：任何代码与配置不得触发真实资金下单。
- 风控阈值：
  - 全程最大回撤 <= 15%
  - 单日最大回撤 <= 5%
  - 单笔亏损 <= 账户总资金 1%
- 触发阈值后必须执行：全仓平仓、暂停交易、进入策略重构流程。
- 合规范围：仅允许交易币安市值前 300 币种。

## 运行基线
- 初始模拟资金：10000 USDT。
- 交易模式环境变量：`TRADING_MODE=SIMULATION`。
- 关键凭据仅允许通过环境变量加载，不得明文入库。

## 核心交付清单
- 交易核心：
  - 币安 API 对接。
  - 模拟撮合引擎（现货、USDT 永续、Alpha 套利）。
  - 自动下单/平仓/持仓管理与异常容错。
- 策略体系：
  - 7x24 行情与绩效监控。
  - 策略更新先回测后上线。
- 前端体系：
  - 实时展示净值、收益/回撤、交易明细、策略状态。
  - 一键全仓平仓、暂停交易入口。
- 运维部署：
  - fly.io 一键部署，Dockerfile 与 fly.toml 标准化。
  - 版本更新与异常回滚能力。
- 监控复盘：
  - 风控与系统告警。
  - 每日交易复盘报告与优化记录。

## 迭代与发布纪律
- 每次程序更新必须：
  - 递增 `VERSION`
  - 更新 `CHANGELOG.md`（中文说明，版本更新时间加上更精准的时间戳）
  - 更新本文件（记录决策、风险、结果）（版本更新时间加上更精准的时间戳）

## 更新记录 v0.9.0 - 2026-04-11
- 重构背景：当前策略毛盈亏 -141.79，胜率 10.2%，80% 退出为分数轮动——生产盈利的能力不足，而非费用问题。
- 核心设计：由“分数轮动”转型为“趋势骑行”，让赢家跑更久，提高单笔赢利量；同时 MA 确认入场提升胜率。
- 关键改动：
  - 决策节流 60s（从 5s）消除噫交易
  - 删除分数轮动退出，改为趋势反转/利润保护/停滞三重退出
  - 永续基准止盈 2.4%→5.0%，止损 1.0%→2.2%，跟踪止损 1.0%→2.0%
  - 新增盈亏平衡锁（+2%后止损锁在入场价上方）
  - 自适应系统降温，新增无交易时自动宽松
- 策略版本升级至 v2，部署时自动重置参数。
- 验证：48 项测试全部通过。
- 风险提示：可能在极端快速轮动行情中入场偏慢，但整体目标是将策略从负期望扰正并持续创新高。

## 更新记录 v0.8.38 - 2026-04-11
- 根因结论：最新线上样本显示策略仍处于负期望，主要由“高分追涨后止损”与“轮动退出后再入抑制失效”共同驱动；此前 4/9 历史还存在大量超短周期轮动噪声交易。
- 关键修复：轮动再入冷却从“完全等值匹配备注”改为“前缀匹配”，`机会衰减退出 score=...` 现可被正确识别并阻断立即再入。
- 策略重构：新增止损隔离（同币种连续止损后进入禁开窗口）和追涨熔断（24h 过热但短周期走弱时拒绝入场），从入场层面切断亏损簇。
- 模型调整：新增 `short_trend` 与 `mean_reversion_risk` 因子，降低“仅靠24h动量追末端”偏差；同时对总分限幅，恢复阈值调参有效性。
- 验证结果：核心回归测试通过（43 项），新增针对性场景测试通过；准备发布后持续观察净值斜率、止损簇数量与手续费/已实现盈亏比。
- 风险提示：重构后交易频率会明显下降，可能牺牲部分强趋势段首轮收益，但优先目标是先把负期望扭转为可稳定优化的正期望。
