# AI项目记忆

## 目标
- 独立于人类版本，构建 AI 自治交易治理实验项目（仅模拟盘）。
- 人类始终具备最高权限和应急控制权。

## 红线
- 仅允许 `SIMULATION`。
- 最大回撤、单日回撤、单笔亏损阈值必须持续监控。
- 凭据只允许环境变量，不允许明文入库。

## 当前版本
- v0.1.14
- 已完成 AI 版与人类版市场池规则对齐：统一使用 Binance 实时市场池，不再硬编码少量交易对。

## 更新记录 v0.1.14 - 2026-04-08
- 核心修复：AI 版市场数据与交易对来源从“固定少量模拟币”升级为“统一实时市场池”，对齐人类版市场规则。
- 对齐内容：
	- 现货/永续：仅 `TRADING` + `USDT`，剔除前20大市值与稳定币基资产。
	- Alpha：仅保留 24h 涨跌幅绝对值 >= 10% 且活跃度达标（24h成交额、24h成交笔数、30日中位成交额）的标的。
- 行为变化：
	- 开仓从统一市场池中选择，仓位按净值 2%-6% 预算计算，不再只交易几个固定币。
	- OHLCV 时序同样来自统一市场池，不再使用硬编码价格区间。
- 风险与结论：
	- 风险：Alpha 接口慢时会拖累市场池刷新。
	- 缓解：已加入 Alpha 扫描上限（40）和刷新频率控制（默认60 ticks），确保主循环稳定。

## 更新记录 v0.1.10 - 2026-04-08
- 问题收口：确认 AI 控制台代码已部署到 Fly 后，追加禁缓存响应头，解决“线上已更新但浏览器仍展示旧模板”的高概率缓存场景。
- 实施方案：首页 `/` 和治理接口 `/api/dashboard`、`/api/ai/governance` 统一返回 `Cache-Control: no-store, no-cache, must-revalidate, max-age=0` 等头部。
- 验证补充：新增首页与治理接口缓存头测试，确保后续改动不会回退到可缓存页面。
- 风险与结论：
	- 极端情况下，用户本地浏览器仍可能保留一次旧标签页内存态；但只要重新请求页面，服务端将明确要求不缓存，恢复成本显著降低。

## 更新记录 v0.1.9 - 2026-04-08
- 修复问题：AI 控制台刷新后仍显示空白占位的问题，根因收敛为首页首屏完全依赖前端脚本拉取数据。
- 实施方案：首页路由改为先获取 `governance_payload`，再由 `dashboard.py` 直接输出完整首屏 HTML，确保浏览器脚本异常时仍可见真实数据。
- 验证补充：新增首页服务端渲染回归测试，覆盖标题、结构化指令中心、无“加载中...”占位和 bootstrap 数据注入。
- 风险与结论：
	- 前端后续软刷新仍依赖浏览器脚本，但即便脚本再次异常，用户也能看到最新一次服务端渲染快照，不会再出现整屏空白。
