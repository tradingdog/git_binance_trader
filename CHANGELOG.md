# 更新日志

## v0.1.0 - 2026-04-04
- 完成安全整改：移除明文凭据，改为环境变量加载。
- 新增 `.env.example` 与 `.gitignore`，避免密钥误提交。
- 建立版本文件 `VERSION`，作为每次更新递增基准。
- 建立项目记忆与执行规范文档，约束后续迭代流程。

## v0.1.1 - 2026-04-04
- 补充核心使命、红线、交付范围与项目专用Agent约束

## v0.2.0 - 2026-04-04
- 新增 FastAPI 后端服务，提供健康检查、仪表盘数据和交易控制接口。
- 新增模拟撮合、风控、动量策略、只读 Binance 行情接入与复盘报告骨架。
- 新增轻量 Web 控制台，支持查看净值、持仓、成交、观察池，并提供暂停交易与一键全仓平仓入口。
- 新增 Dockerfile、fly.toml、pytest 测试与项目说明，完成第一版可运行系统骨架交付。

## v0.3.0 - 2026-04-05
- 新增后台循环执行器，服务启动后可按周期持续运行模拟策略，不再依赖页面请求触发。
- 新增日报自动落盘能力，并在健康检查中暴露最近一次策略执行时间。
- 完成 Fly.io 应用创建与首次部署，线上地址为 `https://git-binance-trader-sim.fly.dev/`。
- 完成线上健康检查验证，当前部署实例运行于 `sin` 区域且保持模拟盘模式。

## v0.4.0 - 2026-04-05
- 策略升级为 `adaptive_opportunity_v1`：引入机会评分、轮动退出、仓位暴露限制与风险预算。
- 撮合引擎新增止损、止盈与跟踪止盈自动平仓逻辑，提升持仓生命周期管理能力。
- 前端切换为观察者模式，隐藏手动暂停/恢复/平仓按钮，仅用于监控策略表现。
- 已完成 Fly 滚动部署并通过线上健康检查验证。

## v0.4.1 - 2026-04-05
- 安全修复：移除 `.env.example` 中误写入的明文 `GITHUB_TOKEN`，恢复为环境变量占位。
- 发布治理：同步递增 `VERSION` 与 Python 包版本号，保持版本一致性。

## v0.5.0 - 2026-04-05
- 新增每小时报告机制：按固定间隔自动生成 `report-YYYYMMDD-HH00.md`，并维护 `latest.md` 快照。
- 新增日志系统：输出 `logs/strategy.log`，记录每轮策略状态、权益、回撤、持仓与策略洞察。
- 新增观察者接口：`/api/reports`、`/api/reports/latest`、`/api/logs/tail`，支持前端展示与运维排查。
- 修正展示口径：报告标题与时间戳改为“每小时策略报告 + 生成时间”，降低时间理解偏差。

## v0.6.0 - 2026-04-05
- 行情接入升级：优先使用真实 Binance API（现货 + 永续 + 上线信息）并自动携带 API Key 请求头。
- Alpha 语义修正：Alpha 视为币安可交易分类/新上市机会，不再作为“套利”语义。
- 策略候选池升级：不再硬性限制前 300 币，扩大到更广泛 USDT 交易对并对 Alpha 机会加权。
- 资金口径修复：新增 `持仓市值` 与 `现金+持仓校验差` 指标，前端可直接核验现金余额是否正确。

## v0.6.1 - 2026-04-05
- Alpha 分类修正：改为直接使用 Binance Alpha 官方接口 `token-list` 与 `get-exchange-info`，不再用推断逻辑判定。
- 持仓键修正：内部改为“市场类型 + 交易对”复合键，避免同名现货/Alpha/永续标的相互覆盖。
- 提交规范修正：后续提交标题统一采用版本号前置格式，例如 `[v0.6.1] feat: ...`。

## v0.7.0 - 2026-04-05
- 观察面板升级：持仓、成交明细、观察池新增市场类型、杠杆、数据来源等字段。
- 资金口径升级：永续仓位改为保证金占用模型，新增 `保证金占用` 指标，现金余额更接近真实风控视角。
- 持仓明细升级：新增止损、止盈显示，便于观察开仓后的跟踪管理。

## v0.8.0 - 2026-04-05
- 前端新增净值曲线图，放置于持仓上方，支持 1 小时、4 小时、日线、周线切换，默认 1 小时。
- 新增净值历史持久化落盘，服务重启后仍可保留最近一周净值轨迹供前端展示。
- 新增存储容量守护：默认要求至少保留 1500MB 可用空间，不足时自动清理旧报告、旧日志与旧净值历史。
- Fly 部署配置新增 2GB 持久卷挂载到 `/data`，报告、日志与净值历史统一写入持久目录。

## v0.8.1 - 2026-04-05
- 行情真实性修复：移除所有伪造回退行情，API 拉取失败时不再生成随机/种子数据，策略本轮跳过交易。
- 价格来源修复：现货与永续分别使用 Binance 各自接口价格，避免“永续标记却取现货价”的串线错误。
- 持仓一致性修复：修复止损/止盈自动平仓键匹配 bug（改为 `market_type:symbol`），避免应平未平。
- 策略一致性修复：同名币种跨市场打分退出改为复合键，且同一币种只保留最佳市场候选，避免重复开仓干扰真实性。
- 新增完整回归测试覆盖上述链路。

## v0.8.2 - 2026-04-05
- 净值曲线升级：补全 X/Y 坐标刻度，新增时间轴与净值轴标签。
- 交互升级：鼠标悬停净值点可显示该点时间和净值，支持十字准星提示。
- 成交明细升级：新增可滚动窗口，默认 500 条，支持切换 1000/2000/5000 条。
- 日志面板升级：页面底部新增日志窗口，默认 500 条，支持切换 1000/2000/5000 条和一键复制。
- 行情拉取稳定性修复：Binance spot/perp/alpha 改为分级容错和重试，alpha 失败不再拖垮主行情链路。

## v0.8.3 - 2026-04-05
- 时间窗修复：1小时/4小时/日线/周线改为严格按当前时间回溯计算，不再出现窗口显示错位。
- 视窗增强：新增横轴拖动查看历史窗口，支持在各时间窗内左右平移回看。
- 联动增强：悬浮净值点时同步高亮对应时间段的成交记录行，便于图表与交易明细联动分析。
- 细节修正：新增“当前窗口时间范围”显示，提升图表阅读可解释性。
